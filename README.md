# mac-setup

Reproducible macOS environment provisioning via Homebrew and shell scripts.

## What It Is

`mac-setup` provides a two-way workflow for managing your macOS development environment in version control. Export your current Mac's configuration, commit it to Git, and apply the same setup to new Macs—ensuring consistency across all your machines.

- **Export**: Snapshot your current Mac's Homebrew packages, dotfiles, and system preferences
- **Version Control**: Track all configuration in Git for reproducibility
- **Restore**: Apply the complete setup to a new Mac with a single script

## How It Works

### The Two-Way Workflow

```
Current Mac (export)          → Git Repository        → New Mac (setup)
┌─────────────────────────┐   ┌──────────────────┐   ┌──────────────────┐
│ ./export_current_mac.sh │──→│ README.md        │──→│ ./setup_new_mac.sh│
│ - Brew packages        │   │ Brewfile         │   │ - Install Xcode  │
│ - Dotfiles             │   │ Dotfiles         │   │ - Install Homebrew
│ - System preferences   │   │ macos-defaults.sh│   │ - Brew bundle    │
└─────────────────────────┘   └──────────────────┘   │ - Symlink config │
                                                      │ - Apply prefs    │
                                                      └──────────────────┘
```

## File Structure

### Scripts

- **`setup_new_mac.sh`** — Idempotent setup script for new machines
  - Installs Xcode Command Line Tools (if missing)
  - Installs Homebrew
  - Runs `brew bundle` to install packages from Brewfile
  - Symlinks dotfiles to home directory
  - Executes `macos-defaults.sh` to apply system preferences
  - Safe guards: checks and skips if already installed

- **`export_current_mac.sh`** — Snapshot your current machine
  - Exports `brew bundle dump` to `Brewfile`
  - Backs up dotfiles (.zshrc, .zprofile, .gitconfig, .gitignore_global, .vimrc)
  - Generates `macos-defaults.sh` template for system preferences
  - Ready to commit and push to Git

### Configuration Files

- **`Brewfile`** — Declarative Homebrew package list
  - Includes: awscli, colima, exiftool, git-lfs, gitsign, k3d, kustomize, podman, terraform
  - Casks: macdown
  - Generated by `brew bundle dump` from your current environment

- **`macos-defaults.sh`** — System UI preferences
  - Key repeat settings
  - Expanded save dialogs
  - Finder extensions visibility
  - Dock hiding preferences
  - Edit freely with personal tweaks

### Dotfiles (Versioned & Symlinked)

- `.zshrc` — Zsh shell configuration
- `.zprofile` — Zsh profile (login shell configuration)
- `.gitconfig` — Git user and core settings
- `.gitignore_global` — Global Git ignore rules
- `.vimrc` — Vim editor configuration

These are symlinked (not copied), so edits in your home directory update the repo and vice versa.

## Requirements

- macOS (Intel or Apple Silicon)
- Git
- Internet connection (for Homebrew downloads)

## Usage

### On Your Current Mac (Export)

1. Clone or navigate to this repository:
   ```bash
   git clone https://github.com/omt-global/mac-setup.git
   cd mac-setup
   ```

2. Export your current configuration:
   ```bash
   ./export_current_mac.sh
   ```
   This creates/updates:
   - `Brewfile` (packages)
   - Dotfiles in the repo root
   - `macos-defaults.sh` template

3. Review and customize `macos-defaults.sh`:
   ```bash
   vim macos-defaults.sh
   # Edit to match your preferences
   ```

4. Commit and push:
   ```bash
   git add .
   git commit -m "snapshot: export current mac setup"
   git push origin main
   ```

### On a New Mac (Setup)

1. Clone the repository:
   ```bash
   git clone https://github.com/omt-global/mac-setup.git
   cd mac-setup
   ```

2. Run the setup script:
   ```bash
   ./setup_new_mac.sh
   ```
   This will:
   - Install Xcode Command Line Tools (interactive prompt if needed)
   - Install Homebrew
   - Install all packages via `brew bundle`
   - Symlink dotfiles
   - Apply system preferences from `macos-defaults.sh`

3. Done! Your new Mac now matches your configuration snapshot.

## Key Features

- **Idempotent**: Safe to run multiple times; skips already-installed components
- **Cross-Platform**: Works on both Intel and Apple Silicon Macs
- **Symlinked Dotfiles**: Keep files version-controlled while using them live
- **Error Handling**: Uses `set -euo pipefail` for strict bash execution
- **Safe Defaults**: Checks before overwriting, no destructive surprises

## Customization

### Add/Remove Packages

Edit `Brewfile` directly or regenerate from your current system:
```bash
brew bundle dump --file=Brewfile --force
```

### Modify System Preferences

Edit `macos-defaults.sh` to add or change defaults. Examples:
```bash
# Set key repeat rate
defaults write NSGlobalDomain KeyRepeat -int 2

# Show hidden files in Finder
defaults write com.apple.finder AppleShowAllFiles -bool true
```

### Add New Dotfiles

1. Place your dotfile in the repo root
2. In `setup_new_mac.sh`, add a symlink line:
   ```bash
   ln -sf "$(pwd)/.my_new_file" "$HOME/.my_new_file"
   ```

## Troubleshooting

- **Homebrew not found**: Ensure `/opt/homebrew/bin` (Apple Silicon) or `/usr/local/bin` (Intel) are in your PATH
- **Permission denied**: Run `chmod +x setup_new_mac.sh` and `chmod +x export_current_mac.sh`
- **Symlink conflicts**: Backup your dotfiles before running `setup_new_mac.sh` if they already exist
